# 软件项目管理
`本项目是开发一个简易的Web应用“图书角”，以此方便班级同学借阅图书。`
## 用例图和类图的作用
用例图作为 UML 建模中 “需求可视化” 的核心工具，在 “图书角” 管理系统项目初始化阶段，主要承担 “明确需求边界、对齐协作认知、指导开发优先级” 的作用，具体适配作业要求如下：1. 精准划分角色功能，匹配 “用户与管理员双角色” 要求。2. 可视化功能逻辑，降低需求沟通成本。3. 支撑开发计划制定，为版本控制奠定基础。<br>
类图作为 “从需求到技术落地的桥梁”，聚焦系统数据结构与业务逻辑关联，直接支撑作业中 “系统初步设计” 与后续开发的衔接，具体作用如下：1. 定义核心数据模型，匹配 “三类核心实体” 要求。2. 明确类间关系，保障业务逻辑正确性。3. 统一技术设计标准，支撑团队协作与版本控制。
